# Patient-Facing AI Tools — Prisma Models

These models back the 4 patient-facing AI features (PRD §7.1). Each model is
tenant-scoped and should be added to `TENANT_SCOPED_MODELS` in
`services/tenant-prisma.ts`.

Also add a new enum `BillExplanationStatus` to `schema.prisma` alongside the
other enums, and extend `NotificationType` with new entries used by these
features.

---

## Feature 1: AI Bill & Insurance Explainer

```prisma
model BillExplanation {
  id            String                 @id @default(uuid())
  invoiceId     String                 @unique
  patientId     String
  language      String                 @default("en") // "en" | "hi"
  content       String                 @db.Text
  status        BillExplanationStatus  @default(DRAFT)
  flaggedItems  Json                   @default("[]") // line items needing attention (e.g. high non-insured costs)
  approvedBy    String?                // User.id of admin/reception
  approvedAt    DateTime?
  sentAt        DateTime?
  createdAt     DateTime               @default(now())
  updatedAt     DateTime               @updatedAt

  tenantId      String?
  tenant        Tenant?                @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  invoice       Invoice                @relation(fields: [invoiceId], references: [id], onDelete: Cascade)
  patient       Patient                @relation(fields: [patientId], references: [id], onDelete: Cascade)

  @@index([patientId])
  @@index([status])
  @@index([tenantId])
  @@map("bill_explanations")
}
```

Add back-relation on existing models:
- `Invoice.billExplanation BillExplanation?`
- `Patient.billExplanations BillExplanation[]`

## Feature 3: Symptom Diary with Trend Insights

```prisma
model SymptomDiaryEntry {
  id             String    @id @default(uuid())
  patientId      String
  symptomDate    DateTime  @db.Date
  entries        Json      @default("[]") // [{ symptom, severity 1-10, notes? }]
  lastAnalysis   Json?     // { trends, followUpRecommended, reasoning } from LLM
  lastAnalysisAt DateTime?
  createdAt      DateTime  @default(now())
  updatedAt      DateTime  @updatedAt

  tenantId       String?
  tenant         Tenant?   @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  patient        Patient   @relation(fields: [patientId], references: [id], onDelete: Cascade)

  @@unique([patientId, symptomDate])
  @@index([patientId, symptomDate])
  @@index([tenantId])
  @@map("symptom_diary_entries")
}
```

Add back-relation on existing models:
- `Patient.symptomDiaryEntries SymptomDiaryEntry[]`

## Feature 4: Chronic-Disease Coaching Agent

```prisma
model ChronicCarePlan {
  id                     String            @id @default(uuid())
  patientId              String
  condition              ChronicCondition
  checkInFrequencyDays   Int               @default(7) // 1, 3, 7
  thresholds             Json              @default("{}") // e.g. {bpSystolic: 160, bgFasting: 200}
  active                 Boolean           @default(true)
  createdBy              String            // User.id of DOCTOR/ADMIN
  createdAt              DateTime          @default(now())
  updatedAt              DateTime          @updatedAt

  tenantId               String?
  tenant                 Tenant?           @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  patient                Patient           @relation(fields: [patientId], references: [id], onDelete: Cascade)
  checkIns               ChronicCareCheckIn[]
  alerts                 ChronicCareAlert[]

  @@index([patientId, active])
  @@index([tenantId])
  @@map("chronic_care_plans")
}

model ChronicCareCheckIn {
  id                   String     @id @default(uuid())
  planId               String
  patientId            String
  loggedAt             DateTime   @default(now())
  responses            Json       @default("{}") // {bp: "140/90", glucose: 180, symptoms: [...]}
  thresholdsBreached   Json?      // null when within thresholds; else list of breached keys + values

  tenantId             String?
  tenant               Tenant?    @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  plan                 ChronicCarePlan @relation(fields: [planId], references: [id], onDelete: Cascade)
  patient              Patient         @relation(fields: [patientId], references: [id], onDelete: Cascade)

  @@index([planId, loggedAt])
  @@index([patientId, loggedAt])
  @@index([tenantId])
  @@map("chronic_care_checkins")
}

model ChronicCareAlert {
  id                String                    @id @default(uuid())
  planId            String
  patientId         String
  severity          ChronicCareAlertSeverity  @default(MEDIUM)
  reason            String                    @db.Text
  createdAt         DateTime                  @default(now())
  acknowledgedBy    String?                   // User.id of doctor
  acknowledgedAt    DateTime?

  tenantId          String?
  tenant            Tenant?                   @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  plan              ChronicCarePlan           @relation(fields: [planId], references: [id], onDelete: Cascade)
  patient           Patient                   @relation(fields: [patientId], references: [id], onDelete: Cascade)

  @@index([planId, createdAt])
  @@index([patientId, acknowledgedAt])
  @@index([tenantId])
  @@map("chronic_care_alerts")
}
```

Add back-relations on existing models:
- `Patient.chronicCarePlans ChronicCarePlan[]`
- `Patient.chronicCareCheckIns ChronicCareCheckIn[]`
- `Patient.chronicCareAlerts ChronicCareAlert[]`
