# Ops-Quality AI Feature — Proposed Prisma Models

Apr 2026 — proposed additions to `packages/db/prisma/schema.prisma` to support the three
ops-quality AI features: Fraud & Anomaly Detection, AI QA on Clinical Documentation,
and Patient Voice & Sentiment Analytics. These models are **not yet added to the schema**
(per caller instruction); copy the blocks below into `schema.prisma` and run
`prisma db push` / `prisma generate` when wiring up persistence.

---

## 1. `FraudAlert` — fraud & anomaly detection signals

```prisma
enum FraudAlertType {
  DUPLICATE_CHARGE
  PRESCRIPTION_OUTLIER   // volume > 3σ above doctor baseline
  HIGH_FREQUENCY_PATIENT
  LARGE_REFUND
  LARGE_DISCOUNT
  GENERIC_TO_BRAND_UPSELL
  OTHER
}

enum FraudAlertSeverity {
  INFO
  SUSPICIOUS
  HIGH_RISK
}

enum FraudAlertStatus {
  OPEN
  ACKNOWLEDGED
  DISMISSED
  ESCALATED
}

model FraudAlert {
  id              String             @id @default(uuid())
  type            FraudAlertType
  severity        FraudAlertSeverity @default(SUSPICIOUS)
  status          FraudAlertStatus   @default(OPEN)
  entityType      String             // "Invoice" | "Payment" | "Prescription" | "Doctor" | "Patient"
  entityId        String
  description     String
  evidence        Json               // shape: { ruleHits: [...], sample: [...], scores: {...}, llmReason?: string }
  detectedAt      DateTime           @default(now())
  acknowledgedBy  String?            // userId
  acknowledgedAt  DateTime?
  resolutionNote  String?

  tenantId        String?
  tenant          Tenant?            @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  @@index([status, severity])
  @@index([type])
  @@index([entityType, entityId])
  @@index([detectedAt])
  @@index([tenantId])
  @@map("fraud_alerts")
}
```

Add `"FraudAlert"` to `TENANT_SCOPED_MODELS` in `apps/api/src/services/tenant-prisma.ts`.

## 2. `DocQAReport` — AI QA audits of clinical documentation

```prisma
model DocQAReport {
  consultationId   String   @id
  score            Int      // 0..100
  issues           Json     // [{ category, severity, description }, ...]
  recommendations  Json     // [string, ...]
  auditedAt        DateTime @default(now())
  auditedBy        String   @default("SYSTEM")

  // Optional sub-scores surfaced by the LLM for display / trending.
  completenessScore Int?
  icdAccuracyScore  Int?
  medicationScore   Int?
  clarityScore      Int?

  tenantId         String?
  tenant           Tenant?   @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  consultation     Consultation @relation(fields: [consultationId], references: [id], onDelete: Cascade)

  @@index([auditedAt])
  @@index([score])
  @@index([tenantId])
  @@map("doc_qa_reports")
}
```

Counterpart on `Consultation`: `docQaReport DocQAReport?`.

Add `"DocQAReport"` to `TENANT_SCOPED_MODELS`.

## 3. `FeedbackSentiment` — AI sentiment/theme extraction for PatientFeedback

Two possible shapes — pick ONE. The separate-table version is preferred so the
`analyzeFeedback` job can fail / rerun without touching the immutable feedback
record.

### 3a. Separate table (preferred)

```prisma
enum SentimentBucket {
  positive
  neutral
  negative
}

model FeedbackSentiment {
  feedbackId       String           @id
  sentiment        SentimentBucket
  emotions         Json             // string[]
  themes           Json             // string[]
  actionableItems  Json             // string[]
  analyzedAt       DateTime         @default(now())

  tenantId         String?
  tenant           Tenant?          @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  feedback         PatientFeedback  @relation(fields: [feedbackId], references: [id], onDelete: Cascade)

  @@index([sentiment])
  @@index([analyzedAt])
  @@index([tenantId])
  @@map("feedback_sentiment")
}
```

Counterpart on `PatientFeedback`: `aiSentiment FeedbackSentiment?`.

Add `"FeedbackSentiment"` to `TENANT_SCOPED_MODELS`.

### 3b. Inline columns (alt.)

Add to existing `PatientFeedback`:

```prisma
aiSentimentBucket   SentimentBucket?
aiEmotions          Json?
aiThemes            Json?
aiActionableItems   Json?
aiAnalyzedAt        DateTime?
```

## 4. `NpsDailyRollup` — daily aggregated NPS-driver snapshot

```prisma
model NpsDailyRollup {
  date                DateTime @id @db.Date   // unique — one row per calendar day
  windowDays          Int      @default(30)
  positiveThemes      Json     // [{ theme, count, sampleQuotes: [...] }, ...]
  negativeThemes      Json     // [{ theme, count, sampleQuotes: [...] }, ...]
  actionableInsights  Json     // [string, ...]
  totalFeedback       Int
  generatedAt         DateTime @default(now())

  tenantId            String?
  tenant              Tenant?  @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  @@index([tenantId])
  @@map("nps_daily_rollup")
}
```

Add `"NpsDailyRollup"` to `TENANT_SCOPED_MODELS`.

## Migration notes

1. Copy enums + models into `packages/db/prisma/schema.prisma` (after the existing
   `PatientFeedback` block is a natural home for 3 & 4; put `FraudAlert` near the
   `Invoice` block; put `DocQAReport` near the `Consultation` block).
2. Update `TENANT_SCOPED_MODELS` in `apps/api/src/services/tenant-prisma.ts`.
3. Run `npx prisma db push` against dev + test DBs, then `npx prisma generate`.
4. The three new routes (`ai-fraud`, `ai-doc-qa`, `ai-sentiment`) ship guarded by
   a `MODELS_AVAILABLE` runtime check — until the models are present they return
   HTTP 503 with a helpful message, so the routes are registrable immediately.
