# Triage & Scribe — Proposed Prisma Model Extensions

Status: **DESIGN ONLY — no schema changes applied.** This document collects the
Prisma fields/models that would cleanly close the Feature 1 (AI Triage) and
Feature 2 (AI Scribe) PRD gaps. The current implementation uses workarounds
that embed/parse structured JSON inside existing `String?` (notes) or `Json?`
(soapDraft) columns. Do **not** edit `packages/db/prisma/schema.prisma` based
on this file until the full plan has been reviewed and a staged migration PR
is ready.

## 1. Appointment.aiSummary (GAP-T2)

**Why.** `POST /api/v1/ai/triage/:sessionId/book` currently stuffs a structured
pre-visit summary (chief complaint, symptom fields, conversation transcript,
confidence, language) into `appointment.notes` as a fenced JSON blob
(`<!-- AI_TRIAGE_SUMMARY_JSON ... -->`) because we can't alter the schema in
this patch. A dedicated column would give:

- Clean reads for the attending doctor's timeline (no string parsing).
- Indexed lookup of triage-sourced appointments.
- Clear separation from the human-editable `notes` column.

**Proposed change.**

```prisma
model Appointment {
  // ... existing fields ...
  aiSummary   Json?   // GAP-T2: pre-visit triage summary (symptomFields +
                     // transcript + confidence + language + sessionId)
  aiSummaryV  Int?    // schema version for the blob shape (start at 1)
}
```

**Shape.**

```json
{
  "version": 1,
  "sessionId": "uuid",
  "capturedAt": "2026-04-23T10:15:00Z",
  "language": "en",
  "confidence": 0.72,
  "chiefComplaint": "Chest pain on exertion",
  "onset": "Sudden",
  "duration": "2 days",
  "severity": 7,
  "location": "Retrosternal",
  "associatedSymptoms": ["shortness of breath"],
  "relevantHistory": "Hypertension",
  "redFlagsNoted": [],
  "transcript": [
    { "role": "assistant", "content": "...", "timestamp": "..." },
    { "role": "user",      "content": "...", "timestamp": "..." }
  ]
}
```

**Migration path.**

1. Add `aiSummary Json?` + `aiSummaryV Int?` (nullable, no default).
2. Backfill job: parse existing `<!-- AI_TRIAGE_SUMMARY_JSON ... -->` blocks
   from `notes` into `aiSummary`. Leave `notes` in place for the short term
   so pre-cutover reads still work.
3. Second migration: strip the fenced JSON from `notes` after all readers have
   moved over.

## 2. AIScribeSession.lastRegenAt + entriesAtLastRegen (GAP-S8)

**Why.** The PRD wants the SOAP draft to refresh every 30–60s instead of on
every utterance (which wasted tokens and produced flicker). The current
implementation workaround stashes `{ lastRegenAt, entriesAtLastRegen }` inside
`soapDraft._meta` — functional but fragile (anyone editing `soapDraft` must
preserve `_meta`).

**Proposed change.**

```prisma
model AIScribeSession {
  // ... existing fields ...
  lastRegenAt          DateTime?
  entriesAtLastRegen   Int?       @default(0)
}
```

**Trigger logic (already implemented against the `_meta` shim).**

- First call (lastRegenAt IS NULL) and transcript.length >= 3 → regenerate.
- Else regenerate when `(now - lastRegenAt) >= 30s` OR
  `transcript.length - entriesAtLastRegen >= 5`.

Once the dedicated columns land, drop the `_meta` read/write in
`apps/api/src/routes/ai-scribe.ts` and set these two columns directly on the
`prisma.aIScribeSession.update({ data: ... })` payload.

## 3. AIScribeSession.consultationLinkedId (future, GAP-S6 helper)

**Why.** GAP-S6 ("Compare to previous visit") currently queries
`Consultation` joined through `Appointment.patientId`. For busy practices that
works but scales poorly — indexing Consultation by patient would require a
join to Appointment on every scribe review. A denormalised `patientId` on
Consultation would make this O(1). Not required for v1; noting it here so it
doesn't get lost.

**Proposed change.**

```prisma
model Consultation {
  // ... existing fields ...
  patientId String?
  patient   Patient? @relation(fields: [patientId], references: [id])

  @@index([patientId, createdAt])
}
```

Backfill from `consultation.appointment.patientId` in a one-shot job.

## 4. AIScribeSession — transcript entry speaker becomes required (GAP-S4)

**Why.** `TranscriptEntry.speaker` is currently typed as the union
`"DOCTOR" | "PATIENT" | "ATTENDANT" | "UNKNOWN"`. GAP-S4 added a
`PATCH /:sessionId/transcript/:index/speaker` endpoint so the doctor can
reassign an utterance after the fact. The `UNKNOWN` variant is now mostly
dead weight — only older rows written before April 2026 still have it.

**Proposed change (zod schema + Prisma enum).**

```prisma
enum ScribeSpeaker {
  DOCTOR
  PATIENT
  ATTENDANT
}
```

Migrate existing `UNKNOWN` rows to `DOCTOR` (safest default — the doctor
will re-tag if needed). Remove `UNKNOWN` from the TypeScript union in
`packages/shared/src/types/ai.ts` and from the Zod enum in
`packages/shared/src/validation/ai.ts`.

## Do NOT

- Do **not** merge any of the above while the schema-change freeze is in
  effect. The workaround paths (fenced JSON in notes, `_meta` key in
  soapDraft) are deliberately conservative so the current feature is shipped
  without a DB migration.
- Do **not** add all four changes in a single migration — split per feature
  so each can be rolled back independently. Proposed order:
  (1) `Appointment.aiSummary` (quickest win), (2) `AIScribeSession.lastRegenAt`,
  (3) `Consultation.patientId`, (4) `ScribeSpeaker` enum.
