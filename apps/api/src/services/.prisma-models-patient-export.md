# Patient Data Export — Prisma Model Proposal

Backs the DPDP Act 2023 right-to-portability feature (PRD §7.x) implemented by
the `patient-data-export` route + service. This model is tenant-scoped and
must be added to `TENANT_SCOPED_MODELS` in `services/tenant-prisma.ts`.

Until this migration lands the service persists/reads rows via
`(prisma as any).patientDataExport` with `// TODO(cast)` comments so the API
can ship ahead of the schema.

---

## Model

```prisma
model PatientDataExport {
  id             String                   @id @default(uuid())
  patientId      String
  format         PatientDataExportFormat
  status         PatientDataExportStatus  @default(QUEUED)
  filePath       String?                  // relative path under uploads/exports/
  fileSize       Int?                     // bytes — null until READY
  errorMessage   String?                  @db.Text
  requestedAt    DateTime                 @default(now())
  startedAt      DateTime?
  readyAt        DateTime?
  downloadedAt   DateTime?                // updated on first successful download

  tenantId       String?
  tenant         Tenant?                  @relation(fields: [tenantId], references: [id], onDelete: SetNull)

  patient        Patient                  @relation(fields: [patientId], references: [id], onDelete: Cascade)

  @@index([patientId, requestedAt])
  @@index([status])
  @@index([tenantId])
  @@map("patient_data_exports")
}
```

Add back-relation on existing models:
- `Patient.dataExports PatientDataExport[]`

## `TENANT_SCOPED_MODELS` additions

Add to the `TENANT_SCOPED_MODELS` Set in
`apps/api/src/services/tenant-prisma.ts`:

- `PatientDataExport`

---

## Rate-limit column (no schema change — application-level)

The per-patient "3 requests per 24h" cap is enforced by counting existing
rows in `PatientDataExport` where `requestedAt > NOW() - 24h`. No dedicated
rate-limit table is needed because the export row itself is the durable
record of a request.

---

## Post-merge steps

1. Run `npx prisma migrate dev --name patient_data_export`
2. Run `npx prisma generate`
3. Update `TENANT_SCOPED_MODELS` set in `tenant-prisma.ts` (add `"PatientDataExport"`)
4. Drop the `(prisma as any).patientDataExport` / `TODO(cast)` escapes in
   `apps/api/src/services/patient-data-export.ts` and
   `apps/api/src/routes/patient-data-export.ts`
5. Re-run `npx tsc --noEmit` and the `patient-data-export.test.ts` integration suite
